python程序运行错误

2018-09-27
Python运行问题汇总
Python

1. 提示“IndentationError: unexpected indent”

原因：

  • 缩进问题。python中空格和tab是不通用的。而且python中代码缩进（indent)会影响代码的逻辑关系，影响执行顺序。

    解决方法：

  • 检查缩进问题，特别注意空格。

2.提示“ValueError: invalid literal for int() with base 10: ‘,’”

原因：

  • int型不能将空字符串转化为整数。说明要转换的字符串中有空的或有字母字符


因为在输入整数数组时，7前有空格。

解决方法：

  • 可以在转换前加入检查字符串内容的语句，判断是否有空或字母字符。

3.提示“TypeError: unsupported operand type(s) for +: ‘int’ and ‘str’”

原因：

  • “类型错误：不支持操作类型为整数和字符串”

  • 这里需要解释的最关键的东西是“+”，“+”在python中有两个作用，一个是数学运算符，是用来在整型、浮点型等数学之间进行加法操作的。另一个是用来进行字符串连接的。所以当你的“+”出现在即有数学运算和字符连接的情况下，计算机根本不知道哪两个是要进行字符串连接，哪两个之间要进行数学运算。（原因解答摘自网络）

    解决方法：

  • 将+前后的数据类型统一，可进行强制转换。

错误代码示例：

i=6
j=3
num=9  
print(i+"+"+j+"+"+num)

正确代码：

i=6
j=3
num=9  
print(str(i)+"+"+str(j)+"+"+str(num))

注意：在python中，进行强制类型转换代码格式

4.提示“TypeError: twoSum() missing 1 required positional argument: ‘target’”

错误原因：

  • 定义的类没有初始化或没有赋值，或者初始化有问题

5.“TypeError: twoSum() takes 2 positional arguments but 3 were given”

错误原因：

  • 类函数调用时参数出问题

错误代码示例：

class Solution():
def twoSum(nums,target): 
    for i in range(len(nums)):
        if nums[i]!=0.1:
            for j in range(len(nums)):
                if (target-nums[i])== nums[j]:
                    print("["+str(i)+","+str(j)+"]")             
                    nums[i]=0.1

nums = [2,7,11,5]
target = 9
a= Solution()
a.twoSum(nums,target)

正确代码：

class Solution():
def twoSum(self,nums,target): 
    for i in range(len(nums)):
        if nums[i]!=0.1:
            for j in range(len(nums)):
                if (target-nums[i])== nums[j]:
                    print("["+str(i)+","+str(j)+"]")             
                    nums[i]=0.1

nums = [2,7,11,5]
target = 9
a= Solution()
a.twoSum(nums,target)

解析：

  • 在类调用的时候，函数定义应为：def twoSum(self,nums,target)
  • 对象调用方法时，通过类的调用，在a.twoSum(nums,target)中，实际上调用的时a.twoSum(a，nums,target)
  • 绑定的方法与非绑定方法
